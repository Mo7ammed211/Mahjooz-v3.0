# 🔐 دليل المصادقة الثنائية (2FA) — Two-Factor Authentication Guide

**الإصدار:** v2.1.0  
**التاريخ:** 22 أبريل 2026  
**الحالة:** ✅ مكتمل وجاهز للاستخدام

---

## 📋 نظرة عامة

تم إضافة **نظام مصادقة ثنائي متقدم (2FA)** إلى منصة محجوز لتعزيز الأمان. هذا النظام يوفر طبقة حماية إضافية عند تسجيل الدخول باستخدام **رموز OTP (One-Time Password)**.

---

## ✨ المميزات الجديدة

### 1. 🔐 نظام OTP (توليد رموز التحقق)
- توليد رموز عشوائية من 6 أرقام
- صلاحية محدودة **10 دقائق** فقط
- 3 محاولات فقط للتحقق
- رسالة خطأ واضحة عند انتهاء الصلاحية

### 2. ✅ صفحة التحقق الثنائي
- واجهة بسيطة وسهلة الاستخدام
- حقل إدخال مركز مع تنسيق خاص
- زر "إعادة إرسال الرمز" للحالات الطارئة
- رسائل تنبيهات واضحة

### 3. ⚙️ صفحة الإعدادات الجديدة
- تفعيل/تعطيل 2FA بسهولة
- عرض معلومات الحساب الشخصية
- تغيير كلمة المرور بأمان
- عرض جلسات نشطة

### 4. 🎨 واجهة مستخدم محسّنة
- زر إعدادات ⚙️ في الشريط العلوي
- Toggle switch أنيق لتفعيل 2FA
- تصميم responsive متوافق مع جميع الأجهزة

---

## 🚀 كيفية الاستخدام

### للعملاء والمستخدمين:

#### **تفعيل المصادقة الثنائية:**

1. اذهب إلى **⚙️ الإعدادات** من الشريط العلوي
2. ابحث عن قسم **🔐 المصادقة الثنائية (2FA)**
3. اضغط على **Toggle Switch** لتفعيلها
4. ستظهر رسالة تأكيد: "✅ تم تفعيل المصادقة الثنائية"

#### **تسجيل الدخول مع 2FA:**

1. أدخل بريدك الإلكتروني وكلمة المرور كالمعتاد
2. ستظهر صفحة **🔐 التحقق الثنائي**
3. سيتم إرسال **رمز OTP من 6 أرقام** (يظهر في Console للاختبار)
4. أدخل الرمز في الحقل المخصص
5. اضغط **تأكيد الرمز** أو اضغط Enter
6. تم! ستصل إلى صفحتك الرئيسية

#### **إذا نسيت الرمز:**
- اضغط على **إعادة إرسال الرمز** لطلب رمز جديد
- عند 3 محاولات خاطئة، ستُرجع إلى صفحة التسجيل

#### **تعطيل المصادقة الثنائية:**
- توجه إلى **⚙️ الإعدادات**
- ابحث عن **🔐 المصادقة الثنائية**
- اضغط على Toggle Switch لتعطيلها

## 🔍 الوظائف الفنية

### الدوال المضافة في `core.js`:

```javascript
// ─── 1. توليد رمز OTP
generateOTP()
// يُرجع: رمز عشوائي من 6 أرقام (مثال: "123456")

// ─── 2. إرسال الرمز إلى المستخدم
sendOTP(uid, email)
// يُخزن الرمز في قاعدة البيانات مع انتهاء صلاحية (10 دقائق)

// ─── 3. التحقق من صحة الرمز
verifyOTP(uid, otp)
// يتحقق من الرمز والصلاحية وعدد المحاولات

// ─── 4. تفعيل/تعطيل 2FA
toggleTwoFA(uid)
// يُبدّل حالة 2FA للمستخدم

// ─── 5. التحقق عند تسجيل الدخول
verify2FA()
// يقدم الرمز المُدخل من المستخدم للتحقق
```

### الصفحات الجديدة:

```javascript
// ─── صفحة التحقق الثنائي
renderOTPVerificationPage()
// واجهة إدخال الرمز

// ─── صفحة الإعدادات والأمان
renderSettingsPage()
// إدارة الأمان و 2FA وكلمة المرور
```

## 💾 تخزين البيانات في Firestore

### جدول `users` — حقول إضافية:

| الحقل | النوع | الوصف |
|-------|-------|--------|
| `twoFAEnabled` | Boolean | حالة تفعيل 2FA |
| `otpCode` | String | رمز OTP الحالي (مؤقت) |
| `otpExpiry` | Timestamp | وقت انتهاء صلاحية الرمز |
| `otpAttempts` | Number | عدد محاولات التحقق |

### مثال على البيانات المخزنة:

```json
{
  "uid": "user123",
  "email": "customer@mahjooz.app",
  "name": "محمد العميل",
  "twoFAEnabled": true,
  "otpCode": "345678",
  "otpExpiry": Timestamp(2026-04-22T10:35:00Z),
  "otpAttempts": 1,
  "createdAt": Timestamp(...),
  "firstLogin": false
}
```

## 🧪 الاختبار

### حسابات تجريبية جاهزة:

```
👑 المدير:         admin@mahjooz.app / admin123
🖥️ الموظف:        staff@mahjooz.app / staff123
🏪 صاحب خدمة:    vendor@mahjooz.app / vendor123
🚗 مندوب:         driver@mahjooz.app / driver123
👤 عميل:          customer@mahjooz.app / customer123
```

### خطوات الاختبار:

1. **اختبر التفعيل:**
   - سجّل دخول بحساب عميل
   - اذهب إلى ⚙️ الإعدادات
   - فعّل 2FA
   - تأكد من الرسالة: "✅ تم تفعيل المصادقة الثنائية"

2. **اختبر تسجيل الدخول:**
   - سجّل خروج
   - اختر "دخول كعميل"
   - ستظهر صفحة التحقق الثنائي
   - انسخ الرمز من Console (أو استخدم الرمز المعروض)
   - الصق الرمز وأكّد

3. **اختبر الأخطاء:**
   - جرّب إدخال رمز خاطئ (3 مرات)
   - جرّب إعادة إرسال الرمز
   - انتظر 10 دقائق واختبر انتهاء الصلاحية

4. **اختبر التعطيل:**
   - عطّل 2FA من الإعدادات
   - سجّل دخول مرة أخرى (بدون 2FA)
